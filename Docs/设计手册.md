## 1. 项目目标与范围

### 1.1 背景

注意力缺陷多动障碍（ADHD）儿童需要**可量化、可重复、趣味化**的注意力训练。本系统依托琶洲实验室 **HybridBCI 单通道脑电头环**，将神经信号映射为游戏输入，针对五类注意力维度设计训练关卡。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **维度覆盖** | 选择性、持续性、转移、分配、抑制五类注意力 |
| **输入统一** | 游戏逻辑仅依赖 `BciInputSnapshot`，不感知具体硬件 |
| **代偿可测** | 无头环时用摄像头 + MediaPipe 完成端到端验证 |
| **数据留存** | 每局会话 CSV 记录，支撑疗效评估与竞赛展示 |
| **儿童友好** | 主菜单大按钮、低惩罚暂停、调试可视化 |

### 1.3 当前版本范围

**已实现：**

- MediaPipe 动作桥接（`Tools/mediapipe_bridge/`）替代 OpenSeeFace 运行时路径
- 主菜单 + 五款可玩 Session
- 三种输入模式（代偿 / 脑机 UDP / 键鼠）
- 代偿视频源选择（摄像头 / 视频文件）
- 双眼屏幕校准 + 调试 Avatar 1:1 面捕
- 会话 CSV、历史计分板
- 开源美术导入管线（可选）

**未实现 / 预留：**

- HybridBCI 官方 SDK 直连（当前为 UDP Transport 占位）
- 专注力由头环实时驱动（代偿下由滚轮模拟）
- 云端账号与远程康复平台

### 1.4 五款游戏一览

| 注意力维度 | 游戏 | 核心 BCI 映射 |
|-----------|------|---------------|
| 选择性注意 | 听音寻宝 | 专注力 → 目标音量；眨眼 → 确认 |
| 持续性注意 | 无尽跑酷者 | 专注力 → 速度；头动 → 变道 |
| 注意力转移 | 指令反转 | 专注力 → 节奏；举手/身体倾 → 相反动作；摇头 → 跳过 |
| 注意力分配 | 双线救援 | 专注力 → 事件间隔；眨眼/点头分轨响应 |
| 注意力抑制 | 红灯停绿灯行 | 专注力 → 掉落速度；眨眼抑制误触 |

各游戏详细说明见 [Docs/Games/](Games/)。

## 2. 用户与使用场景

### 2.1 角色

| 角色 | 行为 |
|------|------|
| **儿童玩家** | 佩戴头环或面对摄像头，完成单局 3–5 分钟训练 |
| **家长/治疗师** | 主菜单选模式、选摄像头、查看历史分数 |
| **开发者/评委** | 键鼠模式演示、调试面板查看原始 BCI 快照 |

### 2.2 典型流程

```
启动 Unity / 构建包
    → MainMenu
    → [代偿] 选择摄像头 / 视频文件
    → [代偿] 可选：屏幕校准（左右红点各 3 秒）
    → 选择输入模式（代偿 / 脑机 / 键鼠）
    → 进入游戏
    → 局末计分 → 写入 CSV → 返回主菜单
```

## 4. BCI 输入层设计

### 4.1 统一快照：`BciInputSnapshot`

所有游戏**只读**该结构体，禁止直接访问 MediaPipe 或 UDP：

```csharp
public struct BciInputSnapshot
{
    public float Focus;           // 0–100 专注力
    public bool Blink;            // 本帧眨眼边沿
    public HeadGesture Head;      // None | Nod | Shake | TurnLeft | TurnRight
    public bool GazeOnScreen;      // 眼球是否在屏内（需校准）
    public Vector2 GazeScreenPos;  // 归一化屏幕坐标，左下 (0,0)
    public bool GazeCalibrated;
    public string RawDebug;
}
```

辅助阈值：`IsLowFocus` (<30)、`IsMidFocus` (40–70)、`IsHighFocus` (>70)。

### 4.2 三种输入模式

| 模式 | Provider | 专注力 | 眨眼 | 头动 |
|------|----------|--------|------|------|
| **代偿** | `CompensationBciInputProvider` | 滚轮手动 | MediaPipe | MediaPipe 头/肩姿态 |
| **键鼠** | `MockBciInputProvider` | 滚轮 | 空格 | W/S/Q/E |
| **脑机** | `HybridBciInputProvider` | UDP 包 | UDP 包 | UDP 包 |

- 模式键：`PlayerPrefs` → `adhd_bci_input_mode`
- 切换 API：`BciInputRouter.SetInputMode(BciInputMode)`

### 4.3 路由器：`BciInputRouter`

```
游戏 Update()
    → router.Current   // 当前模式对应 Provider 的 Poll()
```

游戏 Session 基类 `GameSessionBase` 在 `Update` 中缓存 `Current`，子类读取 `LastInput`。

### 4.4 外部脑机传输

- 接口：`IExternalBciTransport`
- 配置：`Assets/_Project/Config/bci_transport.json` 或 `Resources/bci_transport.json`
- 默认：`127.0.0.1:9876` UDP
- 演示：`Tools/hybridbci_bridge/udp_demo_sender.py`

详见 [BciCompensationDevGuide.md](BciCompensationDevGuide.md)。

## 5. 代偿与面部追踪

### 5.1 设计动机

竞赛开发与临床演示阶段常无实体头环。代偿模式用 **MediaPipe**（面部 + 全身姿态）提取动作，使玩法与正式 BCI 映射一致，仅将「专注力」暂由滚轮代替。

### 5.2 视频源

| 来源 | 存储 | UI |
|------|------|-----|
| USB 摄像头 | `CompensationVideoSourceStore` | 主菜单 `CompensationVideoSourcePanel` |
| 视频文件 | 同上（路径持久化） | 「选择视频」按钮 |

摄像头枚举通过 `Tools/mediapipe_bridge/tracker.py --list-cameras`（OpenCV `CAP_DSHOW`）解析（`MediaPipeBridgeLauncher` / `CompensationCameraEnumerator`）。

**记忆（PlayerPrefs）：** 下次启动自动恢复上次配置（含采集分辨率与帧率，见 `CompensationTrackerCaptureSettings`）。

| 键 | 内容 |
|----|------|
| `adhd_comp_capture_kind` | 摄像头 / 视频文件模式 |
| `adhd_comp_camera_index` | 摄像头索引 |
| `adhd_comp_camera_name` | 摄像头设备名（插拔后按名称匹配） |
| `adhd_comp_video_path` | 视频文件路径 |
| `adhd_comp_cap_width/height/fps` | 摄像头采集画面与帧率 |

**眨眼帧率提示：** 采集帧率 **&lt; 18fps** 时 UI 显示警告，眨眼边沿可能漏检；卡顿时请先用「省资源」预设（640×360@15）或降低分辨率。

`BciInputRouter.RestoreSavedMode()` 与主菜单视频源面板加载时均会 `ApplyStoredVideoSource()`；代偿模式下自动重启追踪。

### 5.3 MediaPipe 集成

- Python 桥接：`Tools/mediapipe_bridge/`（`tracker.py` + `face_pose.py` + `body_gestures.py`）
- Unity 客户端：`Assets/_Project/Scripts/Core/MediaPipe/`
- 默认 Python：`EasyVtuber` 嵌入式解释器（`AppRoot` Inspector 可改）
- 依赖：`pip install -r Tools/mediapipe_bridge/requirements.txt`
- 传输：UDP `127.0.0.1:9878` JSON 帧 → `MediaPipeMotionClient`
- 面部算法移植自 **EasyVtuberStudio** `get_pose`（眨眼、头姿、虹膜注视比）
- 身体：`BlazePose` 肩线/脖颈 + 举手（腕高于肩）+ 身体左右倾
- 预览：`CompensationBciInputProvider.showTrackerPreview` 开启 `--preview` OpenCV 窗口

### 5.4 信号映射（代偿）

| BCI 信号 | MediaPipe 来源 |
|----------|----------------|
| 眨眼 | `face_pose.get_pose` 眼睑开合 |
| 点头 | 俯仰角超过阈值 |
| 摇头 | 偏航角振荡 |
| 左右转头 | 偏航角单侧阈值 |
| 眼球 | 虹膜比 → `GazeScreenMapper` → `GazeScreenPos` |
| 举手 | `body_gestures` 腕高于同侧肩（边沿触发） |
| 身体倾 | 肩线水平偏移 `BodyLean` ±1 |

### 5.5 调试 Avatar

`DebugPanel` 内 `InputMotionAvatarView`：

- 使用 `MediaPipeAvatarPoseSolver` 将面部/肩颈关键点转为头/颈/肩骨骼旋转
- **镜像**显示（与摄像头左右一致，参考 EVTS 行为）
- 打开调试窗时重置姿态校准基准

## 6. 眼球屏幕校准

### 6.1 目的

将 MediaPipe 虹膜注视比映射到**物理屏幕坐标**，供选择性注意等玩法扩展「注视点」判定。

### 6.2 流程（`GazeCalibrationController`）

1. 用户于主菜单选择 **代偿** 模式，点击 **屏幕校准**
2. 全屏黑色遮罩 `GazeCalibrationOverlay` 显示**红色校准点**
3. **左侧**：屏幕水平 25%、垂直 50%，注视 **3 秒**，取最后 **1 秒**平均眼球样本
4. **右侧**：水平 75%、垂直 50%，同样 3 秒 / 末 1 秒平均
5. 写入 `GazeCalibrationStore`（PlayerPrefs），`GazeScreenMapper` 标记 `IsCalibrated`

### 6.3 映射算法（`GazeScreenMapper`）

- 左右样本中点 → `GazeCenter`
- 左右样本水平差 → `GazeSpan.x`（对应 50% 屏宽）
- 实时：`screenX = 0.5 + (gaze.x - center.x) / span.x`
- `GazeOnScreen`：归一化坐标落在 [0,1]²

### 6.4 数据流

```
MediaPipe tracker.py (UDP :9878)
    → MediaPipeMotionClient
    → GazeScreenMapper.TryMapToScreen
    → BciInputSnapshot.GazeScreenPos / GazeOnScreen
```

### 7.2 听音寻宝（选择性注意）

详见 [Games/听音寻宝.md](Games/听音寻宝.md)。

**训练目标：** 在混合音效中识别目标声并抑制干扰。

| 要素 | 设计 |
|------|------|
| 专注力 | 调节目标声道音量/清晰度 |
| 眨眼 | 确认「听到目标」 |
| 反馈 | 正确 +分，误报 -分 |
| 开源参考 | [ChrisBrooksbank/attention](https://github.com/ChrisBrooksbank/attention)（范式）；**代码自研** |

### 7.3 无尽跑酷者（持续性注意）

详见 [Games/无尽跑酷者.md](Games/无尽跑酷者.md)。

**训练目标：** 长时间维持跑酷节奏与变道反应。

| 要素 | 设计 |
|------|------|
| 专注力 | 跑速与得分倍率；过低触发暂停提示 |
| 头动 | 左/右转道 |
| 反馈 | 距离计分、障碍碰撞扣分 |
| 开源参考 | [Hira-Runner](https://github.com/Ruaneri-Portela/Hira-Runner)（范式）；美术 [AwesomeRunner](https://github.com/VladimirPirozhenko/AwesomeRunner)（MIT，可选导入） |

### 7.4 指令反转（注意力转移）

详见 [Games/指令反转.md](Games/指令反转.md)。

**训练目标：** 根据规则在「执行 / 反转」间切换认知集。

| 要素 | 设计 |
|------|------|
| 专注力 | 指令出现间隔 |
| 点头 | 确认完成 |
| 摇头 | 表示无法完成 / 跳过 |
| 开源参考 | [StroopApp](https://github.com/MylanBeghin/StroopApp)（MIT，任务切换）；**代码自研** |

### 7.5 双线救援（注意力分配）

详见 [Games/双线救援.md](Games/双线救援.md)。

**训练目标：** 左右视野并行监控，不同动作响应不同事件。

| 要素 | 设计 |
|------|------|
| 专注力 | 左右事件生成间隔 |
| 眨眼 | 左侧救援 |
| 点头 | 右侧救援 |
| 开源参考 | [track-of-thought](https://github.com/macie-k/track-of-thought)；**代码自研** |

### 7.6 红灯停绿灯行（注意力抑制）

详见 [Games/红灯停绿灯行.md](Games/红灯停绿灯行.md)。

**训练目标：** Go/No-Go — 对「晶石」执行动作，对「陨石」抑制动作。

| 要素 | 设计 |
|------|------|
| 专注力 | 物件下落速度 |
| 眨眼 | 收集晶石（绿灯） |
| 抑制 | 陨石出现时**不眨眼** |
| 摇头 | 清除陨石（每局限次） |
| 开源参考 | [response-inhibition-game](https://github.com/jpickavance/response-inhibition-game-unity-project)（Unity + 可选美术导入） |

### 7.7 游戏 Session 基类

`GameSessionBase` 统一：

- 读取 `BciInputRouter`
- 低专注暂停横幅（`AppHudController`）
- `EndSession()` → `SessionRecordService.Append`
- ESC / 局末返回 `MainMenu`

子类实现：`OnSessionStart`、`OnSessionUpdate`、计分逻辑。

## 9. 数据与会话记录

### 9.1 记录模型：`GameSessionRecord`

| 字段 | 说明 |
|------|------|
| Timestamp | UTC 本地时间字符串 |
| GameId / GameNameCn | 游戏标识 |
| InputMode | 代偿 / 脑机 / 键鼠 |
| Score | 本局得分 |
| Correct / Wrong | 正确 / 错误次数 |
| AvgFocus | 平均专注力 |
| PauseCount | 低专注暂停次数 |
| DurationSec | 局时长 |

### 9.2 存储

- 路径：`Application.persistentDataPath/session_records.csv`  
  Windows 示例：`%USERPROFILE%\AppData\LocalLow\<Company>\<Product>\session_records.csv`
- 服务：`SessionRecordService`（`AppRoot` 子组件）
- 主菜单 `ScoreboardPanel` 读取最近 N 条展示

### 9.3 隐私

当前为**本地单机 CSV**，无上传。正式部署需告知监护人数据用途并符合医疗软件合规要求。

## 15. 已知限制与路线图

### 15.1 已知限制

| 限制 | 说明 |
|------|------|
| 专注力代偿 | 代偿模式下专注力非生理信号，需头环或 UDP |
| dshowcapture | 部分 DirectShow 插件未随仓库分发；摄像头列表走 exe 枚举 |
| 美术完整度 | 需手动执行 OSS 导入才有完整 Prefab |
| 单机 CSV | 无多用户与云端报表 |

### 15.2 路线图

| 阶段 | 内容 |
|------|------|
| **P0** | HybridBCI 实机联调，专注力闭环 |
| **P1** | 听音寻宝 AudioMixer 完整混合音效 |
| **P2** | 注视点玩法接入 `GazeScreenPos` |
| **P3** | 治疗师仪表盘 / 疗程计划 |
| **P4** | 移动端或 WebGL 展示版（去 Python/MediaPipe 依赖） |
